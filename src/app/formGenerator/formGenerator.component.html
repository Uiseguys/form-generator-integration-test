Form Generator

READ ME

The HTML structure of the Json Schema Form is the following:

                        
                            	<my-dynamic-form [schema]="schema" [form]="form" [ajv]="ajv">
                                		<my-input for="integer"></my-input>
                                		<my-input for="string"></my-input>
                                		<my-input for="object"></my-input>
                                		<my-dropdown for="array"></my-dropdown>
                                		<my-checkbox for="boolean"></my-checkbox>
                            	</my-dynamic-form>

The root <my-dynamic-form /> accepts 3 props:

  • [schema]: Form definition object.
  • [form]: Form object.
  • [ajv]: JSON Schema Validator. https://github.com/epoberezkin/ajv

Schema Object:

The most important and mandatory properties of the object are the "type" and "$id" property.

"$id" must be unique for every property.

"type" accepts values like: string, integer, boolean, array and object.

Any type, except { "type": "object" } can have properties like:

  • "labelContent" => Labeling an input.
  • "description" => An explanation about the purpose of the instance.
  • "placeholder" => Input placeholder.
  • "default"
  • "examples" => "examples" is an array of strings where you can offer example of values that the input accepts.

Types:

  • { "type": "string" } => The string type is used for strings of text. It may contain Unicode characters.
  • { "type": "integer" } => Used for any integers type numbers.
  • { "type": "boolean" } => true or false values. Used for checkbox type inputs.
  • { "type": "array" } => Accepts only "$id" and "items" properties. "items" is an object of "type" string or integer. Used for checkbox type inputs. Used for dropdown type inputs.
  • { "type": "object" } => Requires a mandatory property, "properties". The properties (key-value pairs) on an object are defined using the properties keyword. The value of properties is an object, where each key is the name of a property and each value is a JSON schema used to validate that property.

    For example, let’s say we want to define a simple schema for an address made up of a number, street name and street type:

                                    
                                    {
                                        	"type": "object",
                                        	"properties": {
                                            		"number":      { "type": "number" },
                                            		"street_name": { "type": "string" },
                                            		"street_type": { "type": "string" }
                                        	}
                                    }

    By default, the properties defined by the properties keyword are not required. However, one can provide a list of required properties using the required keyword.

    The required keyword takes an array of one or more strings. Each of these strings must be unique.

    In the following example schema defining a user record, we require that each user has a name and e-mail address, but we don’t mind if they don’t provide their address or telephone number:

                                    
                                    {
                                        	"type": "object",
                                        	"properties": {
                                            		"name":      { "type": "string" },
                                            		"email":     { "type": "string" },
                                            		"address":   { "type": "string" },
                                            		"telephone": { "type": "string" }
                                        	},
                                        	"required": ["name", "email"]
                                    }

Form Object:

The actual form object that respects the schema.

The inputs can have predefined values or be empty and filled in by the user or dynamicly within the code.

The following Form Object is used for this example:

                        
                            }
                                "checked": false,
                                "duration": }
                                    "min": 5,
                                    "max": 10
                                },
                                "date": }
                                    "dateValue": moment_(new Date()).format('Do MMMM YYYY')
                                },
                                "startDate": "2007-08-31T16:47+00:00",
                                "endDate": "2007-08-31T16:47+00:00",
                                "sources": [
                                    "source1",
                                    "source2"
                                ]
                            }
